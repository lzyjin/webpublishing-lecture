변수 사용하기 - 장점과 네이밍

변수를 사용한 경우와 그렇지 않은 경우의 차이
1. 변수를 사용하면 css 속성 값의 수정 보완이 필요한 경우 한방에 처리할 수 있다.
2. 변수를 사용하면 개별 수정이 되지 않지만 프로젝트 관리 측면에서 일관성 있는 관리가 용이하다.

변수를 사용하면 색상 값과 패딩 값의 수정이 필요한 경우, 변수명에 담긴 색상 값과 패딩 값을 변경하면 변수로 적용된 모든 선택자에 한번에 적용된다.
신경쓰자) 변수명의 제약은 크게 없지만 가독성을 높이기 위해 2개의 단어가 합쳐진 경우 대시 또는 언더바를 통해 구분지어주는 것이 좋다.
변수명이 대시, 언더바를 제외한 특수문자 또는 숫자로 시작하면 안된다.
그리고 속성과 값을 직관적으로 표현하는 단어를 사용하는 것이 바람직하다.

예시로 제목의 색상을 변수로 지정한다면...
heading-color => $heading-color
만약 변수명이 길어서 줄인다면...
heading-font color => $heading-fc
heading-backgroundcolor => $heading-bgc
$h-c 와 같이 이니셜만 사용하지 않습니다 

    예시) 

    $container-color: #000;
    $heading-color:#333;
    $sub-heading-color: #555;
    $heading-padding: 10px;

    .container {
      color: $container-color;
      .items {
        h1 {
          color: $heading-color;
          padding: $heading-padding;
        }
        h2 {
          color: $sub-heading-color;
        }
      }
    }
